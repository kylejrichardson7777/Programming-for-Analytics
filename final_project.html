In this project I will be analzying a dataset of Dungeons and Dragons character sheets from DndBeyond, the offical online interface for the game. Dungeons and Dragons(D&D) is a popular tabletop roleplaying game that boasts millions of players globaly. I found this dataset on kaggle

In D&D, each player has a character sheet which tracks the mechanics and logistics of their character. This sheet is either kept in paper or stored on digital services like DndBeyond. The dataset I'm working with here is from their website and contains over 1.2 million unique character sheets. Analysing this data is incredibly useful for discovering trends between players, and uncovering what are the most popular aspects of the game and other details that could drive marketing or product development.

My research question is: Every playgroup plays D&D in their own unique way. By analyzing the types of characters players'create, can we identify popular styles of play that could be catered to with future D&D products.

From this look, we can quite alot of null values in "Class_Other" and "Subclass_Starting". I next want to see if i differentiate between rows where this value in purposefully left blank and where it is missing.

In D&D, when you create a character you must first pick a class, which you get 1 level in("Class_Starting"). As you progress in the game, you can increase the level of your class. Once a character reaches level 3 you must choose a subclass ("Subclass_starting") for your character. Therefore, you would expect null values in "Subclass_starting" where the character hasn't hit level 3 yet, and conversely, any character with "class_starting">3 and "subclass_starting" is null must be missing data. However in dnd you also have the ability to multiclass ("class_other") - meaning you can level up in a second class at every level up checkpoint. The same subclass rules apply, therefore if Total_Level >= 5 then there must be a value in "Subclass_Starting" OR "Subclass_Other". The below code is an attempt to discover cases where there is true missing data in these columns.

Coefficent correlation is based on mean of x and mean of y, and how they vary. With more data, these values converge quickly. To prove this, we can check the standard error of correlation (1 - r) / (n - 2), where r=0.3(moderate corelation), and n=50,000 0.004. This means that with a sample size of 50k, we can get the true value with an error rate of 0.004%, which is bascially negligble. With this, we can create graphs using a random sample of 50k that will represent the entire dataset and run quickly.

After doing some more domain research, I determined that these stats represent physical prowess and are predominately used in the combat portions of D&D. Likewise the stats which represent mental & social prowess ("Intelligence", "Wisdom", and "Charisma"), which are used in the roleplaying aspect of the game are quite low across our dataset's most popular classes.

From this insight it seems that a good chunk of the players enjoy the combat portion of D&D. With this knowledge, if I was a D&D writer or designer, I could publish more adventures and products that focus on combat. Conversely, there is room for improvement across the mental & social stats where I could perhaps write new classes, subclasses, or adventures which emphasis and promote these aspects of the game.
